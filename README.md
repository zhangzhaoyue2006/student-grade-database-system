# 学生成绩数据库系统

本项目是“数据库及实现”课程期末项目，目标是设计并实现一个用于管理学生成绩的数据库系统。系统基于 Flask + MySQL 开发，支持学生、院系、专业、课程、教师、授课、选课、成绩、统计分析和用户账号等模块的管理与查询。

## 一、项目功能

系统按照管理员、教师、学生三类角色划分权限。

### 1. 管理员端

- 院系管理：新增、修改、删除、查询院系信息。
- 专业管理：新增、修改、删除、查询专业信息，并维护不同学位层次的学分要求。
- 课程管理：新增、修改、删除、查询课程信息，维护课程编号、课程名称、课程说明、学时、学分、学位等级和开课院系等基础信息。
- 教师管理：新增、修改、删除、查询教师信息，维护教师职称。
- 学生管理：新增、修改、删除、查询学生信息，维护学号、主修专业、辅修专业、年级、学位等级等信息，所属院系由主修专业确定。
- 授课管理：管理各学期的课程授课安排，设置任课教师、授课角色和结课状态，并支持按教师、课程、学期和课程状态查询。
- 选课管理：管理学生当前有效的选课记录，添加记录表示学生选课，删除记录表示学生退课或取消选课，删除操作由系统日志记录。
- 统计分析：按学期、开课院系筛选课程，并查看单门课程的成绩统计结果。
- 用户与系统管理：维护用户账号、角色绑定、启用/停用、重置密码、操作日志等。

### 2. 教师端

- 我的授课课程与学生名单：按学期查看授课课程，并进入课程学生名单页面。
- 成绩录入与查询：只能对未结课课程录入或修改成绩；已结课课程只能查询。
- 课程成绩统计：按学期选择本人授课课程，查看平均分、中位数、最高分、最低分、及格率、优秀率和分数段分布；未结课课程统计基于当前已录入成绩。

### 3. 学生端

- 个人信息查询：查看本人基本信息、专业、院系、学位层次等。
- 我的选课查询：选择学期后，系统展示本人该学期的有效选课课程，并统计该学期选课学分和累计已修学分。
- 我的成绩查询：按学期查看已结课课程的最终有效成绩；已结课但无成绩的课程显示为缺考，若存在补考或重修成绩，则取该课程最高分作为最终有效成绩。

## 二、技术栈

- 后端：Python、Flask
- 数据库：MySQL
- 数据库访问：PyMySQL
- 前端：HTML、CSS、JavaScript、Jinja2 模板
- 密码处理：Werkzeug password hash
- 配置管理：python-dotenv

## 四、环境要求

运行本项目需要提前安装：

- Python 3.10 或以上版本
- MySQL 8.0 或以上版本
- Windows PowerShell

建议使用虚拟环境安装 Python 依赖，避免和本机其他 Python 项目冲突。

## 五、安装与启动

### 1. 获取项目代码

如果从 GitHub 克隆仓库：

```powershell
git clone <你的仓库地址>
cd student-grade-database-system
```

如果已经下载到本地，直接进入项目根目录即可。

### 2. 创建并启用虚拟环境

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

### 3. 安装依赖

```powershell
.\.venv\Scripts\pip.exe install -r requirements.txt
```

### 4. 配置环境变量

复制 `env.example` 并重命名为 `.env`，然后按照本机 MySQL 配置填写数据库密码。

`env.example` 内容如下：

```text
SECRET_KEY=student-grade-system-dev-key
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=student_grade_system
```

其中 `DB_PASSWORD` 需要改成本机 MySQL 对应用户的密码。

### 5. 初始化数据库

确认 MySQL 服务已经启动，然后在项目根目录执行：

```powershell
database\init_db.bat
```

脚本会依次执行：

- `database/schema.sql`：创建数据库和数据表。
- `database/seed.sql`：写入测试数据和默认账号。

执行过程中会提示输入 MySQL root 密码。

### 6. 启动 Web 系统

```powershell
.\.venv\Scripts\python.exe run_dev.py
```

启动成功后，本机浏览器访问：

```text
http://127.0.0.1:5001
```

如果需要同一局域网内其他设备访问，需要确认防火墙允许 5001 端口访问，然后使用本机 IPv4 地址访问，例如：

```text
http://本机IPv4地址:5001
```

本机 IPv4 地址可以通过以下命令查看：

```powershell
ipconfig
```

## 八、主要业务规则

- 管理员可以维护基础数据和账号信息，但不能录入或修改学生成绩。
- 教师只能管理自己授课课程的成绩。
- 未结课课程允许教师录入或修改成绩；已结课课程只允许查询成绩。
- 学生只能查看自己的个人信息、选课信息和成绩信息。
- 选课表只保存当前有效选课记录；添加记录表示学生选课，删除记录表示退课或取消选课，相关删除操作由系统日志记录。
- 已修学分根据已结课且成绩合格的课程自动计算；补考和重修合格后计入学分，重复课程只计算一次。
- 系统使用参数化 SQL 查询，避免用户输入直接拼接进 SQL 语句，降低 SQL 注入风险。
